**Log server endpoints:**
```
POST /log     — Extension POST single entry
POST /batch   — Extension POST multiple entries
GET  /logs    — Read recent logs (?count=N)
GET  /health  — Health check
GET  /clear   — Clear logs
```

**Log sources:** [BG] = background.js, [CS] = content-script.js

**Key log markers untuk saya cek:**
- `[TOKEN] OK` → token berhasil di-capture
- `[TOKEN] ERROR` → token gagal → cek root cause
- `[CONVS] OK` → API conversations berhasil
- `[CONVS] API returned error` → API fails → cek HMAC atau token
- `[SEND] Sent ✅` → message berhasil dikirim
- `[SEND] Failed` → message gagal → cek API response
- `[POLL] FAILED — no token` → token capture issue
- `[SW] Tevi CS Bot v0.1.0 started` → SW restart

# =========================================
# POLLING FLOW (MV3)
# =========================================

1. User klik "Aktifkan" → chrome.alarms.create
2. Alarm fire (every 3 min):
   a. chrome.scripting.executeScript → tab Tevi → getTeviToken()
   b. POST log [TOKEN] OK / ERROR
   c. GET conversations with HMAC signed request
   d. POST log [CONVS] result
   e. For each conv: classify → decide action
   f. SEND reply → POST log [SEND] Sent ✅ / Failed
   g. Update chrome.storage.local state
3. POST log [POLL] Done summary

# =========================================
# DEBUGGING CHECKLIST
# =========================================

Setiap kali ada issue, cek dalam urutan:

1. **Token** → `GET /logs` → cari `[TOKEN]`
   - [TOKEN] OK → token capture working
   - [TOKEN] ERROR / no token → content script issue

2. **API** → `GET /logs` → cari `[CONVS]`
   - [CONVS] OK → API accessible
   - [CONVS] API returned error → HMAC issue atau token expired

3. **Send** → `GET /logs` → cari `[SEND]`
   - [SEND] Sent ✅ → reply delivered
   - [SEND] Failed → API error

4. **SW restart** → cari `[SW]`
   - SW sering restart di MV3 → normal behaviour
   - Jika banyak restart → SW keep-alive issue

5. **File log size** → jika >1MB → run GET /clear

# =========================================
# MV3 SERVICE WORKER NOTES
# =========================================

- importScripts() TIDAK bisa load ES module → semua inline
- chrome.scripting.executeScript untuk programmatic injection
- chrome.alarms untuk polling (bukan setTimeout — itu tidak reliable di SW)
- SW di-kill setelah ~30s idle → chrome.alarms keep it alive
- fetch() dari SW context perlu semua headers (Origin, Referer, Accept)
- localhost fetch ALLOWED dari extension tanpa special permissions
